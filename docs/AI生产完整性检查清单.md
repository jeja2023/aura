# AI 生产完整性检查清单

本文用于寓瞳项目当前 AI 能力（特征提取、向量入库、向量检索、重试补偿、聚类支撑）的上线前验收。

## 一、配置完整性（必须通过）

- [ ] `Ai:BaseUrl` 已指向正确 AI 服务地址（与部署环境一致）。
- [ ] AI 服务模型路径已配置，启动后健康检查返回 `model_loaded=true`。
- [ ] 生产环境已确认 ArangoDB 可用（若要求持久化索引，禁止仅内存模式）。
- [ ] `.NET` 与 AI 的接口鉴权一致（已配置 `Ai:ApiKey` 与 `X-Aura-Ai-Key`）。
- [ ] `ARANGO_URI/ARANGO_DB/ARANGO_USER/ARANGO_PASSWORD` 均为真实值，非占位值。
- [ ] 限流与超时参数已评估：`HttpClients:Ai:*` 不低于实际推理耗时需求。
- [ ] 图片与元数据大小限制已确认：`Limits:MaxImageBase64Chars`、`Limits:MaxMetadataJsonChars`。
- [ ] `.env` 与环境变量管理合规，未将真实密钥提交到仓库。

## 二、功能链路完整性（必须通过）

- [ ] `POST /ai/extract` 可稳定返回 512 维向量。
- [ ] `POST /ai/extract-file` 在文件路径模式下可稳定提取特征。
- [ ] `POST /ai/upsert` 可写入向量索引，返回成功且 `vid` 可复查。
- [ ] `POST /ai/search` 能按相似度返回 TopK 结果，分数排序正确。
- [ ] 后端 `api/vector/extract`、`api/vector/search` 与 AI 服务联调通过。
- [ ] 抓拍链路中 AI 失败时可入重试队列，重试成功后可补写元数据与向量。
- [ ] 重试超过阈值后处理符合预期（重试次数、失败审计、清理策略）。
- [ ] `POST /ai/cluster` 可执行特征聚类并返回簇统计与成员结果。

## 三、性能容量完整性（建议通过）

- [ ] 单张图片提特征 P95 耗时满足目标（建议记录 CPU 与 GPU 两套基线）。
- [ ] 批处理参数 `_BATCH_SIZE`、`_MAX_WAIT_SECONDS` 已根据真实流量压测调整。
- [ ] 吞吐压测已覆盖峰值场景（持续输入、突发输入、混合大小图片）。
- [ ] 向量检索在目标数据规模下响应可接受（TopK 查询延迟满足 SLA）。
- [ ] API 重试与熔断未造成雪崩放大（失败场景下仍可控）。
- [ ] 资源占用可控（CPU、内存、显存、线程数无持续异常增长）。

## 四、容灾降级完整性（必须通过）

- [ ] AI 服务不可用时，后端表现符合预期（错误码、提示、审计日志完整）。
- [ ] ArangoDB 短暂故障时，降级策略符合环境要求（开发可降级、生产按策略拒绝）。
- [ ] AI 调用失败不会阻断抓拍主链路入库（允许后补偿）。
- [ ] 重试队列积压可观测、可人工触发处理，避免长期无感堆积。
- [ ] 服务重启后索引一致性已验证（内存索引与持久化策略差异已知晓）。
- [ ] 就绪检查可识别关键依赖异常（AI 服务、模型、数据库、缓存、密钥）。

## 五、精度与业务有效性（建议通过）

- [ ] 已构建最小验收样本集（同人多时段、跨摄像头、不同光照/遮挡）。
- [ ] 已评估检索质量指标（Top1/Top5 命中率或等价业务指标）。
- [ ] 相似度阈值有业务解释，并经过现场样本验证。
- [ ] 误识别、漏识别对业务流程的影响已评估并有兜底策略。
- [ ] 聚类结果（若启用）与业务认知一致，退化策略可接受。

## 六、安全与合规完整性（必须通过）

- [ ] AI 服务对外访问已受控（内网、网关、ACL、鉴权头至少满足其一）。
- [ ] 非必要接口不暴露公网，文档接口在生产环境按需关闭或限制。
- [ ] 抓拍图片存储路径与访问策略符合最小权限原则。
- [ ] 关键操作具备审计日志（提取失败、重试、入库、检索异常）。
- [ ] 敏感配置（密钥、口令、连接串）已纳入统一密管策略。
