# 运维上线手册

本文合并原部署手册、上线检查清单、readiness 说明与 AI 生产检查清单，作为当前版本上线与日常巡检的统一入口。

## 1. 部署入口

- Docker 编排：`docker/docker-compose.yml`
- Docker 环境模板：`docker/.env.docker.example`
- 启动：`docker/up.ps1` 或 `docker/up.sh`
- 停止：`docker/down.ps1` 或 `docker/down.sh`
- 健康检查：`docker/check.ps1` 或 `docker/check.sh`

上线前复制模板：

```powershell
Copy-Item docker\.env.docker.example .env.docker
```

然后替换 `.env.docker` 中的镜像标签、端口、数据库密码、JWT/HMAC 密钥与管理员密码。

## 2. 基础服务

统一 Compose 会拉起：

- PostgreSQL：业务主库
- Redis：缓存、限流、重试队列、SignalR backplane
- ArangoDB：AI 向量/索引存储
- AI：FastAPI 特征提取与检索服务
- API：.NET 中枢服务
- `arango-init`：确保 `ARANGO_DB` 数据库存在
- `db-migrate`：执行 PostgreSQL 增量迁移

首次部署服务器可以临时连接互联网，将 `IMAGE_PULL_POLICY` 设为 `missing` 并使用 `docker/up.ps1 -Build` 或 `docker/up.sh --build` 完成镜像拉取/构建/启动。部署成功后断开互联网，并改回 `IMAGE_PULL_POLICY=never`。

## 3. 上线前检查

- [ ] `.env.docker` 中所有 `REPLACE_*` 占位值已替换。
- [ ] `AllowedHosts` 已配置实际域名或局域网 IP。
- [ ] `JWT__KEY` 与 `SECURITY__HMACSECRET` 为足够长的随机值。
- [ ] `POSTGRES_PASSWORD`、`ARANGO_ROOT_PASSWORD`、`AURA_ADMIN_PASSWORD` 不使用默认值。
- [ ] `models/osnet_ibn_x1_0.onnx` 存在，或 `AURA_MODEL_HOST_DIR` 已指向正确目录。
- [ ] `IMAGE_PULL_POLICY` 与实际镜像来源一致。
- [ ] API、AI、PostgreSQL、Redis、ArangoDB 端口未被占用。
- [ ] 临时联网部署期间使用安全组/防火墙限制访问来源，部署完成后按要求断开互联网。
- [ ] 如启用 `AURA_API_KEY`，API 与 AI 使用同一个值。
- [ ] 如启用 Prometheus 或 OpenTelemetry，仅允许内网监控系统访问相关端点。

## 4. 健康与就绪

基础健康检查：

```powershell
powershell -ExecutionPolicy Bypass -File .\docker\check.ps1
```

脚本会检查：

- AI `/live`
- AI `/ready`
- API `/api/health`

业务就绪检查使用：

- `GET /api/ops/readiness`
- 需要超级管理员登录
- 用于检查 JWT、HMAC、AI、告警通知、数据库等关键配置

## 5. AI 生产检查

- [ ] `AURA_MODEL_PATH` 指向容器内模型路径。
- [ ] `GET /ready` 返回 `model_loaded=true`。
- [ ] ArangoDB 可连接，`ARANGO_DB` 存在。
- [ ] 检索接口限流与队列参数符合现场并发。
- [ ] 生产环境默认关闭详细内部错误输出，必要时短时开启排障。
- [ ] 已执行 AI 巡检：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\ops\aura-ops.ps1 ai-check
```

## 6. 回归与压测

统一入口：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\ops\aura-ops.ps1 readiness
powershell -ExecutionPolicy Bypass -File .\scripts\ops\aura-ops.ps1 capture-regression
powershell -ExecutionPolicy Bypass -File .\scripts\ops\aura-ops.ps1 full-check
```

其中：

- `readiness`：上线就绪检查
- `capture-regression`：抓拍链路回归
- `full-check`：全系统联调与压测

## 7. 数据与停机

`docker/down.*` 默认保留命名卷，数据不会丢失。只有确认需要清空 PostgreSQL、Redis、ArangoDB 与 API storage 数据时，才使用：

```powershell
powershell -ExecutionPolicy Bypass -File .\docker\down.ps1 -Volumes
```

该操作会删除命名卷，生产环境慎用。

## 8. 断网后的离线更新

在联网构建机执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\docker\build-images.ps1
powershell -ExecutionPolicy Bypass -File .\docker\offline-pack.ps1
```

将 `docker/dist/aura-offline-<时间戳>/` 上传到已断网服务器后执行：

```powershell
docker load -i aura-images.tar
docker compose --env-file .env.docker -f docker-compose.yml up -d --no-build
```

默认保留命名卷，更新不会清空数据库和运行数据。
