# 抓拍链路端到端测试清单

## 一、测试目标

验证“抓拍接入 -> AI提取 -> 向量写入/检索 -> 告警与日志 -> 重试队列”的全链路可用性与稳定性。

## 二、环境准备

- 后端服务已启动：`backend/Aura.Api`
- AI服务已启动：`ai/main.py`
- MySQL 可用并已导入 `database/schema.sql`
- Redis 可用（用于重试队列）

## 三、用例清单

- [ ] 登录获取 Token 成功（`/api/auth/login`）
- [ ] 海康抓拍接入成功（`/api/capture/push`，验签通过）
- [ ] ONVIF 抓拍接入成功（`/api/capture/onvif`）
- [ ] C++ SDK 抓拍接入成功（`/api/capture/sdk`）
- [ ] 抓拍记录可查询（`/api/capture/list`）
- [ ] AI 提取成功，`metadata_json` 含 `ai_success/ai_dim/ai_msg`
- [ ] 向量检索接口可返回结果（`/api/vector/search`）
- [ ] 元数据含“异常”时自动生成告警（`/api/alert/list`）
- [ ] 操作日志有完整记录（`/api/operation/list`）
- [ ] AI 失败时进入重试队列（`/api/retry/status`）
- [ ] 重试处理后成功/失败计数正确（`/api/retry/process`）

## 四、判定标准

- 核心接口响应 `code = 0`。
- 抓拍、告警、日志三类数据可互相追溯（时间、设备、通道一致）。
- 重试队列长度和处理结果与预期一致。

## 五、回归建议

- 每次调整适配器、AI客户端、缓存/队列逻辑后执行一次完整回归。
- 至少覆盖海康、ONVIF、C++SDK 三种接入路径。
