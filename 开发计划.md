# 寓瞳 - 智能集宿区视觉解析与人员追踪平台 · 研发全景计划

## 一、项目核心愿景

聚焦**集宿区/公寓/园区**安防场景，构建端到端的视频图像智能解析平台。

**核心数据链路**：接收边缘 NVR 的高质量抓拍 → AI 深层特征提取与聚类 → 以**楼层平面图**和**多边形防区（ROI）**构建精细物理空间映射 → 最终还原人员的**落脚宿舍**、**漫游轨迹连线**及**社交关系图谱**，同时具备完善的企业级管控底座。

> 系统适配独立封锁的安防视频物理网段，打通后可通过 JSON 单向输出通道，向内网大数据平台低损耗供数。

---

## 二、核心技术栈与架构定型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **接入网关** | .NET Core WebAPI | 适配器架构：内置海康 ISAPI / C++ SDK / ONVIF 通用协议三通道，支持多品牌 NVR 扩展 |
| **交互前端** | 原生 Vanilla JS + Three.js | **3D主视界 + 2D业务切片** 混合架构，Canvas 独立绘图，部署于 .NET `wwwroot` |
| **中枢服务** | .NET Core (C#) + Kestrel | 内置 **SignalR** 实现低延迟双工推送（大屏红点/热力等） |
| **AI推理引擎** | Python (FastAPI) | 内网独立算力沙盒，接收图片，输出 512 维特征向量 |
| **主务数据库** | PostgreSQL 16+ | 空间多边形研判、层级权限结构、业务数据持久化 |
| **特征向量库** | ArangoDB | 海量人脸/人体特征高维检索 |
| **缓存与消息** | Redis | 防阻塞缓冲、死信队列回收、热数据加速 |

## 三、系统数据流架构

```
┌─────────────┐                     ┌──────────────────────────────────┐
│  NVR 设备群  │                     │     .NET 接入网关 (适配器层)      │
│             │     各品牌协议       │                                  │
│ • 海康威视   │ ──────────────────► │  ┌─ 海康 ISAPI 适配器            │
│ • 大华      │                     │  ├─ C++ SDK 适配器              │
│ • 宇视      │                     │  ├─ ONVIF 通用适配器             │
│ • 其他品牌   │                     │  └─ (可扩展...)                  │
└─────────────┘                     │         │                        │
                                    │    ICaptureAdapter 统一输出      │
                                    │    (标准抓拍数据结构)             │
                                    └────────┬─────────────────────────┘
                                             │ 抓拍图+元数据
                                             ▼
                         ┌───────────────────────────────┐
                         │      .NET 中枢服务 (C#)        │
                         │  • 图片预处理 & 入库           │
                         │  • 业务调度 & 研判引擎          │
                         │  • SignalR 双工推送            │
                         └──┬────────────┬────────────┬──┘
                            │            │            │
                   ┌────────▼──┐  ┌──────▼─────┐  ┌──▼──────────┐
                   │ Python AI │  │ PostgreSQL  │  │  ArangoDB   │
                   │ (FastAPI) │  │  主务数据库  │  │  特征向量库  │
                   └────────┬──┘  └──────┬─────┘  └──┬──────────┘
                            │            │            │
                            └────────────┼────────────┘
                                         │
                                  ┌──────▼──────┐
                                  │    Redis    │
                                  │  缓存/消息  │
                                  └─────────────┘
                                         │
                         ┌───────────────────────────────┐
                         │     前端 (Vanilla JS/Three.js)  │
                         │  • 3D 楼宇数字孪生外壳          │
                         │  • 2D 楼层平面图 & ROI 编辑     │
                         │  • 轨迹回放 & 图谱展示         │
                         └───────────────────────────────┘
```

> **适配器核心设计**：所有品牌的 NVR 数据通过各自的适配器（实现 `ICaptureAdapter` 接口），统一转换为**标准抓拍数据结构**（图片二进制 + 时间戳 + 设备通道号 + 元数据 JSON）。后续的 AI 推理、空间碰撞、业务研判等所有环节完全不感知数据来源品牌，实现真正的设备无关性。

## 四、核心功能模块全景

### 模块一：安全认证与鉴权中枢

| 功能项 | 说明 |
|--------|------|
| 用户登录认证 | 支持严苛密码策略，基于 `JWT` 会话令牌，登录后所有请求强制校验令牌 |
| 设备级 IP 白名单 | NVR 接入接口启用 IP 放行池 + HMAC 签名验证，防止伪造数据注入 |

### 模块二：RBAC 权限组织与用户管理

| 功能项 | 说明 |
|--------|------|
| 三级角色权限管理 | 自定义角色（超级管理员、楼栋管理员、安保巡查员等），角色绑定路由级权限开关 |
| 集宿物理资源树 | 管理 "园区 → 楼栋 → 楼层 → 房间" 的多层级从属映射字典档 |

### 模块三：空间配置与地图编辑

| 功能项 | 说明 |
|--------|------|
| 楼层图纸管理 | 上传楼层 CAD 平面图，缩放布置摄像头物理点位（X, Y 坐标标定） |
| ROI 多边形编辑器 | Canvas 绘制多边形防区，框定每个摄像头视野中对应的房间门/区域并入库 |

> **ROI 核心逻辑**：每个摄像头可定义多个多边形 ROI，每个 ROI 映射到物理资源树中的具体房间。当 AI 检测到人体出现在某 ROI 区域内时，等同于判定该人员进入对应房间。多边形顶点以 JSON 数组存储于 PostgreSQL。

### 模块四：态势感知实况大屏

| 功能项 | 说明 |
|--------|------|
| 实时 3D 告警主视界 | 引入 Three.js 渲染轻量级 3D 建筑白模外壳，发生高优告警时，对应楼栋/楼梯/楼层闪烁 3D 立体红光 |
| 2D 切片空间下发 | 在 3D 界面点击特定楼层，拉出 2D 平面切片图进行精准落点查看（避开 3D 空间碰撞矩阵计算成本） |
| 楼层热力感知打点 | 有人过镜时，大屏 2D 切片图对应摄像头位置闪烁呼吸光晕，直观感知实时动态 |

### 模块五：历史追踪与轨迹回溯

| 功能项 | 说明 |
|--------|------|
| 以图搜轨引擎 | 上传模糊截图，经 ArangoDB 向量检索，关联该人员的虚拟 ID (`Virtual_Person_ID`) 及全部过镜记录 |
| 平面轨迹动画播放 | 按时间序列在 2D 平面图上绘制带箭头的历史动线，高保真还原完整行动路径 |

### 模块六：智能研判与异常告警

| 功能项 | 说明 |
|--------|------|
| 落脚宿舍自动研判 | 每日定时执行，根据 ROI 停留时长与频次，自动推断人员最终归属房间 |
| 群租/异常滞留告警 | 单人间 ROI 内出现多个不同 ID 且停留超时，触发群租或外人滞留告警 |
| 夜不归宿考勤告警 | 至归寝截止时间仍未捕获到人员进入归属楼层 ROI，生成考勤异常名单 |

### 模块七：统计驾驶舱与报表导出

| 功能项 | 说明 |
|--------|------|
| 数据看板 | 基于 ECharts 展示留宿率、异常占比等宏观数据图表 |
| 报表导出 | 群租/晚归/轨迹明细等列表支持 `.xlsx` / `.csv` 格式下载 |

### 模块八：审计日志与系统监控

| 功能项 | 说明 |
|--------|------|
| 业务操作日志 | 记录所有管理人员操作：登录、下载报告、修改配置等，全行为可溯源 |
| 系统运行日志 | 记录 NVR 连接状态、AI 推理异常、消息队列回收等系统级运行状况，支持界面查看 |

## 六、关键 API 接口规划

### 6.1 NVR 接入接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/capture/push` | 海康 ISAPI Webhook 推送入口（含 HMAC 验签） |
| POST | `/api/capture/sdk` | C++ SDK 通道抓拍推送 |
| POST | `/api/capture/onvif` | ONVIF 通用协议抓拍推送（适配大华/宇视等品牌） |
| GET | `/api/device/list` | 查询已注册 NVR 设备列表（含品牌、协议、在线状态） |
| POST | `/api/device/register` | 注册新 NVR 设备（指定品牌与协议类型） |

### 6.2 空间配置接口

| 方法 | 路径 | 说明 |
|------|------|------|
| CRUD | `/api/campus/*` | 园区/楼栋/楼层/房间资源树管理 |
| CRUD | `/api/floor/*` | 楼层图纸上传与管理 |
| CRUD | `/api/camera/*` | 摄像头布点管理 |
| CRUD | `/api/roi/*` | ROI 多边形防区管理 |

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/ai/extract` | 图片特征提取（返回 512 维向量） |
| POST | `/ai/search` | 以图搜人（ArangoDB 向量检索） |

### 6.4 业务查询接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/track/{vid}` | 查询虚拟人员轨迹 |

| GET | `/api/alert/list` | 查询告警列表 |
| GET | `/api/stats/overview` | 统计驾驶舱概览数据 |

### 6.5 外部输出接口

| 方法 | 路径 | 说明 |
|------|------|------|

## 八、非功能性需求

### 8.1 性能指标

| 指标 | 目标值 |
|------|--------|
| 单张抓拍处理延迟 | ≤ 2 秒（含 AI 推理） |
| ArangoDB 特征检索响应 | ≤ 500 毫秒（百万级库） |
| SignalR 推送延迟 | ≤ 200 毫秒 |
| 大屏页面首次加载 | ≤ 3 秒 |
| 系统并发承载 | ≥ 50 路摄像头同时推送 |

### 8.2 可靠性

- 抓拍数据推送失败时自动进入 Redis 死信队列，支持自动重试与人工干预
- AI 推理服务异常时中枢服务自动降级，不影响其他功能正常运行
- PostgreSQL 主从备份策略，关键数据定期增量备份

### 8.3 安全合规

- 所有 API 通信强制 HTTPS（内网可配置为 HTTP）
- 密码存储使用 BCrypt 单向哈希
- 操作日志不可删除，仅支持查询
- 图片存储文件夹强制目录隔离，防止路径穿透
- 前端鉴权 Token 策略：已完成前端 `localStorage` token 清理与 `Authorization` 头移除，统一走 HttpOnly Cookie 会话；CSP 已支持按环境配置并可按域名/端口持续收紧白名单
- 抓拍 HMAC 签名策略：已在生产环境强制设备级密钥优先，设备未配置密钥时不再回退全局密钥（开发环境保留联调回退）
- SignalR 鉴权粒度：`EventHub` 已升级为 `[Authorize(Policy="楼栋管理员")]`，并完成角色分组推送替换（由 `Clients.All` 收敛为角色组广播）
- AI/重试链路大对象传输：已落地“禁止内联 Base64 回退入队”策略（生产默认禁用），并完成主链路/重试链路优先按文件路径提特征，进一步压缩 Base64 传输面
- 图片落盘一致性：已补齐抓拍成功场景图片归档（`Storage:SaveCaptureImageOnSuccess`），并写入 `capture_record.image_path`
- 可观测性：数据库/缓存关键链路结构化日志已覆盖，后台任务已补执行耗时日志；代码层已提供告警通知抽象与 Webhook/文件双通道，运维阶段仅需配置真实通知地址与接收端
- 后台任务生命周期：每日研判已迁移到标准 `BackgroundService` 托管模式并保留分布式锁防重，满足并发部署与生命周期治理要求
- 开发配置与示例账号：代码库侧弱凭据/默认密码与生产弱参数风险已完成清理与 Fail-Fast 校验；开发一键启动脚本已增加配置预检，配置中心与密钥托管为运维环境落地项
- 运维就绪性：已新增后端 `readiness` 接口用于集中检查关键配置就绪状态；前端历史 token 兼容壳逻辑已清理，避免后续偏离 Cookie 会话安全基线
- 运维就绪性：已补充 `readiness` 运维使用说明文档（接口调用、字段解释、上线前检查步骤、失败处理），可直接纳入上线流程与发布前阻断规则

## 十、风险识别与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| NVR 型号差异导致接入协议不兼容 | 部分设备无法接入 | 适配器模式隔离差异，优先适配海康 ISAPI，次选 ONVIF 通用协议兜底 |
| 多品牌 NVR 数据格式差异大 | 适配器开发工作量增加 | 定义严格的 `ICaptureAdapter` 统一输出标准，各品牌只需实现接口即可接入 |
| 非主流品牌缺少 ONVIF 完整实现 | 个别设备功能受限 | 提供手动上传抓拍图片的降级方案，确保业务不中断 |
| AI 聚类漂移导致同一人分裂为多个 V_ID | 研判准确率下降 | 引入人工合并机制 + 定期重聚类 |
| 摄像头画面遮挡/角度不佳影响 ROI 判定准确性 | 归寝误判 | 多摄像头交叉验证 + 置信度阈值可配置 |
| 大规模并发抓拍导致处理队列积压 | 实时性下降 | Redis 队列削峰 + AI 引擎水平扩容 |
| 楼层平面图格式不统一 | 地图展示异常 | 统一转换为 PNG/SVG 标准格式后上传 |

## 十一、竞品分析与差异化定位

### 11.1 市场竞品分类对比

#### 第一类：NVR 厂商自带智能平台

| 竞品 | 核心能力 | 局限性 |
|------|---------|--------|
| **海康威视** iVMS-8700 / HikCentral | 人脸抓拍、以图搜图、考勤打卡、事件回溯 | 深度绑定海康硬件生态，无法接入其他品牌 NVR；空间拓扑能力弱，无自定义 ROI 多边形研判 |
| **大华** DSS Pro / ICC 平台 | 人脸识别、行为分析、门禁考勤 | 同样绑定大华设备，跨品牌不可用；集宿区归寝/群租等垂直场景功能缺失 |
| **宇视** IMOS 平台 | 视频监控统一管理、基础智能分析 | 侧重设备运维管理，业务研判与空间分析能力薄弱 |

#### 第二类：第三方视频 AI 中台

| 竞品 | 核心能力 | 局限性 |
|------|---------|--------|
| **商汤** SenseFoundry 方舟 | 城市级视频 AI 中台，人脸/人体/车辆全域分析 | 面向城市/大型园区，部署成本极高（百万级起），集宿区场景过度设计 |
| **旷视** 盘古/河图 | 园区安防 + 物流机器人 | 偏向智慧楼宇门禁和仓储物流，非集宿区专项 |

| **虹软** ArcFace SDK | 高精度离线人脸算法引擎 | 仅提供算法 SDK，无上层业务系统，需完全自行开发 |

### 11.3 核心差异化优势

**1. 设备无关性**
- 适配器架构（`ICaptureAdapter`）打破厂商锁定，海康/大华/宇视/ONVIF 设备均可接入
- 新增品牌只需实现适配器接口，核心业务代码零改动

**2. 场景纵深度**
- 专攻集宿区/公寓/园区的**被动无感知**归寝分析，无需人员主动配合
- 群租检测、夜不归宿告警、异常滞留预警等功能为行业独有

**3. 空间智能引擎**
- ROI 多边形防区 + 楼层平面图拓扑是核心技术壁垒
- 将摄像头视野精确映射到物理房间，实现"看到即定位"

**4. 成本可控**
- 全开源技术栈（.NET Core + Python + PostgreSQL + ArangoDB + Redis），无商业 AI 平台授权费
- 适配现有 NVR 设备，无需额外采购专用硬件

**5. 安全合规**
- 支持纯内网物理隔离部署，适配安防视频专网要求
- 对外仅输出 JSON 结构化数据，原始图片不出网
