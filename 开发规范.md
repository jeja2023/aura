# 寓瞳系统开发规范与约束机制

为了确保《开发计划.md》中所述的架构和全景功能高质量落地，并保证代码及其架构的长期可维护性，特制定本开发规范。所有参与本项目的研发人员均须严格遵守以下核心约束。

---

## 摘要原则

1. **中文主导**：语言、编码、注释、日志、响应、文档全面中文化。
2. **结构清晰**：分门别类、简短表意、拒绝同名。
3. **极简整洁**：URL精简无冗杂，禁用开发调试残留。
4. **谨慎修改**：最小化改动解决问题，严禁破坏性重构与遗漏。
5. **前端全局复用**：如果发现没有全局样式，必须先创建统一的全局样式和组件类再进行复用。
6. **双主题支持**：强制使用 CSS 变量（CSS Variables）实现暗黑/浅色的双主题自适应，拒绝色彩硬编码。

---

## 1. 语言与字符规范

*   **全中文环境**：系统提示、接口响应、反馈文案等，始终使用**简体中文**进行回复和处理。
*   **统一编码**：所有的代码和各类说明文件，在创建和修改时**必须采用 UTF-8 等支持中文的统一编码**，坚决杜绝出现中文字符乱码现象。
*   **文档规范**：说明类等非代码文档，其文件命名需尽量使用中文，使其直观易懂（如 `开发规范.md`、`部署说明.md`）。

## 2. 目录文件结构与命名规范

*   **分类存放**：编写项目时，禁止代码扁平化随意堆砌。必须按照功能层面或架构（如接口层、服务层、缓存层、数据映射层）将文件进行**分门别类存放**，确保目录结构高度清晰。
*   **简短表意**：各类代码文件的命名必须力求**简短**，不需要过长，但必须**能够准确体现文件的核心作用**。
*   **拒绝全局同名**：整个项目源码结构内，即使位于不同层级的包或目录下，**业务及逻辑文件也尽可能不要出现同名**，防止全局搜索或重构时导致混淆或引发严重的人为覆盖事故。

## 3. Web 路由及 URL 规范

*   **保持简短直接**：API 路由和页面 URL 路径设计必须紧贴业务，缩短层级，力求简练。
*   **剔除废弃词缀**：**严禁在任何环境的 URL 和路由中出现诸如 `backend`、`frontend`、`html`、`php` 等多余甚至暴露底层技术栈的无用前或后缀。**

## 4. 编码与注释规范

*   **中文注释与提示**：要求所有代码块、业务接口、核心逻辑处均使用**中文注释**以及良好的中文提示。
*   **清理开发痕迹**：在代码合入基线之前，**不得在代码库中遗留任何“开发调试性语言”（如生硬随意测试打印、`// 测试用的、TODO等无意义内容` 等）**，保持代码整体纯净。

## 5. 日志与异常记录规范

*   **中文日志落盘**：不论是系统报错日志、数据库操作日志、服务运行状态还是针对外部 API（如 AI 接口或 NVR 推送）的回调记录，系统输出日志**必须使用中文全方位记录**动作执行情况，便于排查人员快速定位。

## 6. 代码修改及维护约束

在进行 Bug 修复、逻辑优化或功能增强时，必须秉持**“对现有稳定部分始终抱有敬畏之心”**的原则：

*   **极简干预**：修改代码务必谨慎小心，在确切保证其他已有功能不被影响的前提下，以**最小代价、最小改动范围**解决当前问题。
*   **严禁盲目重命名**：不要随意触碰、篡改上下文原有的变量名和方法名，除非该命名本身存在词不达意的历史错误。
*   **禁止功能丢失**：对现有模块进行逻辑修改或重构替换时，**千万不要遗漏任何旧代码中承载的关键判断与正常功能**。修改完成后必须进行充分的回归自测验证。

## 7. 前端样式与组件复用规范 (UI 及交互统一)

为了保障系统整体视觉与体验的高度统一以及长远维护性，前端界面的编写必须绝对遵循“**复用大于新建**”的思想：

*   **绝对全局复用及统一管理**：不论是系统中的各类控件，包括但不限于**按钮、数据表格（Grid/Table）、输入框、搜索框、模态框（Modal/Dialog）、全局提示（Toast/Alert/Message）**等，必须全量、统一使用项目中已封装的全局样式与公共组件库。
*   **按钮统一规范（强制）**：业务按钮优先使用全局类：主要动作使用 `.btn-primary`，次要动作使用 `.btn-secondary`。禁止在页面私有 CSS 中单独改按钮圆角、边框、主色和悬浮态。
*   **文件选择按钮统一规范（强制）**：`input[type="file"]` 的“选择文件”按钮必须复用全局样式（`::file-selector-button`/`::-webkit-file-upload-button`），禁止保留浏览器默认按钮样式导致页面风格不一致。
*   **文件选择控件宽度统一规范（强制）**：业务页横排卡片内的文件输入禁止用 `flex:1` 占满整行；统一使用主题变量 `--form-file-input-basis` / `--form-file-input-min` / `--form-file-input-max`（在 `theme.css` 定义，由 `forms.css` 生效）。特殊页面如需更窄/更宽，须在页面 CSS 中**仅覆盖上述变量或写更高优先级选择器**，不得在各页复制一套互不一致的宽高。
*   **前端第三方脚本禁止 CDN（强制）**：Three.js、SignalR、ECharts 等浏览器端库必须放在 `frontend/common/vendor/` 并由现有 `*-vendor-loader.js` 或等价方式**仅引用本地路径**；禁止在 HTML/JS 中写死 `jsDelivr`、`unpkg` 等外链地址，避免内网与合规环境不可用。
*   **无样式先创建，严禁硬编码**：当现有的设计库或公共 CSS 未匹配到某个页面所需的新交互或新样式时，**绝不允许开发人员在页面的 `.html`, `.aspx` 或 `.vue/jsx` 中直接使用内联（inline）或者是独立的页面私有包裹样式拼凑**。你必须：
    1. 前往存放核心公共 CSS/基础组件库的地方提取并**优先创建一整套全局生效的样式类或封装组件**。
    2. 等基础资源创建完毕后，再去具体的业务页面通过调用 Class Name 或组件标签来进行**复用**。以保障将来主题修改时只需更改一处即可全局生效。
*   **页面资源一一对应（强制）**：每个页面都必须创建独立的 `css` 与 `js` 文件，并与页面文件一一对应。命名保持同名且尽量短，优先单词命名。例如：`login.html` 必须对应 `login.css`、`login.js`；`stats.html` 必须对应 `stats.css`、`stats.js`。
*   **页面目录同名归档（强制）**：每个页面的 `html/css/js` 必须放入与页面同名的独立目录中。例如：`login/login.html`、`login/login.css`、`login/login.js`；`track/track.html`、`track/track.css`、`track/track.js`。
*   **文件头双语注释（强制）**：每个页面文件顶部必须添加“中文名称 + 英文名称”的文件头注释。示例：HTML 使用 `<!-- 文件：登录页（login.html） | File: Login Page -->`，CSS/JS 使用 `/* 文件：登录样式（login.css） | File: Login Styles */`、`/* 文件：登录脚本（login.js） | File: Login Script */`。

## 8. 双主题色彩系统规范 (Light / Dark Theme)

寓瞳系统既包含日常配置运维页面，也包涵了用于 3D 楼台与轨迹映射监控的态势大屏。前端系统**必须基于 CSS 变量强制支持“浅色（Light）”与“暗黑（Dark）”双主题模式的完美切换**。

*   **杜绝颜色词硬编码**：整个工程中的组件背景底色、文字色、边框、状态图标颜色等，**坚决不允许直接写死具体的 HEX 值或色号 (如 `color: #333`、`background: #0B0E14`)**。无论是使用哪种技术栈，均必须通过 CSS 变量（如 `var(--bg-main)`、`var(--text-color)`）调用对应的体系色度池。
*   **全局色彩变量盘约束**：
    *   **主品牌色 (Primary)**：Light 和 Dark 下统一使用科技视觉的高亮蓝/青色，供各种主搜索按钮和表单高亮状态使用。
    *   **背景体系 (Background)**：
        *   **Light 模式**：背景采纯净白 / 浅灰板色系设计（`#F0F2F5`/`#FFFFFF` 等），确立最精细标准的阅读观感。
        *   **Dark 模式**：采取极夜深灰黑面板（`#0B0E14`/`#141B26`），为 3D 大屏展现提供悬浮感底色托盘。
    *   **多阶反馈色 (Status)**：
        *   **高危红 (Danger)**：如群租、断联、极端删除等，统一采用霓虹警示红（`--color-danger`）。
        *   **运行绿 (Success)**：如抓拍流解析正常、提交完成（`--color-success`）。
        *   **异常橙 (Warning)**：如夜间告警临近等（`--color-warning`）。
    *   **文字反色 (Text)**：须随主题环境实现正负黑白梯度的色值反转（如 Light 深灰变 Dark 亮灰及纯白等）。

未来任何开发新增的新型弹层、数据 Grid 列或是浮现边框卡片，全部强制向该 CSS Theme Variables 双体系看齐，确保开发风格的持续生命力。
